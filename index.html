What is Responsive Web Design ?
Итан Маркот написал статью в журнал A List Apart в 2010 году в которой описал ключевые идеи формирующие адаптивный дизайн. Он предложил использовать такие техники как flexible grid layout, flexible images добавил media query и назвал это все RWD. Надо заметить что RWD это набор техник и идей, новый подход в разработке веб сайтов..
Идея состоит в том, что ваш сайт должен смотреться хорошо (красиво) на любом устройстве. Сайт должен как-бы отвечать и подстраиваться под девайс, меняясь (как хамелеон) так, чтобы всегда выглядеть удобно для пользователя. При этом мы создаем только одну версию сайта
Потому что устройств с которых пользователи заходят в интернет и смотрят сайты становиться все больше и больше и они все разные, очень сложно и дорого писать отдельный сайт для каждого устройства. RWD это новый подход, и своего рода "сдвиг" в привычных нам подходах создания сайтов. Следуя этой идее можно сделать сайт который будет выглядеть хорошо на существующих девайсах и на девайсах которых еще нет (харпктеристики которых мы еще не знаем)
Давайте разделим эти два понятия, mobile дизайн - проектируется для девайсов, например если надо использовать фичи устройства. А Responsive - для контента, чтоб контент всегда выглдел хорошо и был доступен
Так сложилось, что мы создаем 2 версии сайтов - для мобильных устройств и для десктопных компьютеров. В ситуации когда ты смотришь сайт с телефони и хочешь поделиться классной ссылкой с другом - отправляя ее как m.mysite.com/ha-ha друг откроя ее на компьютере ужаснется. Используя RWD можно разрабатывать одну версию сайта
Пользователи не должны адаптироваться к твоему сайту, сайт должен адаптироваться к пользователям.
Эта фраза Брюса Ли отлично описывает идеологию RWD
В нашем мире пользователи уже сейчас серфят сайты с множества разнообразных девайсов, используя RWD подход мы можем больше не беспокоится о том как сайт будет смотреться на каждом конкретном устройстве из множества доступных.

Fluid Grids
• Fluid layouts - layouts that scale.
• Instead of based on pixels, your layouts are percentage based.
• Allows your site to adapt to screen sizes in between the ones you’re targeting

Margin & Padding
• Margins: context is the width of the element’s container.
• Padding: context is the width of the element itself.
Начните с пикселей и применяйте волшебную формулу, о которой далее получите %
How to find %
• Begin with a pixel based layout. These pixels will become the basis for your percentage calculations.
• Divide the target pixel value by the pixel value of the parent. Don’t forget margins and padding.
• Add a comment of your original calculation. (this will save you)

Flexible Images
• Images are not given a defined height and width in the HTML
• Images are sized based on the width of it’s parent container
• Images are defined with a width of 100% in the CSS.
• This also applies to video, embed and object tags
